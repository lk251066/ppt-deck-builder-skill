- Agenda or Path
- Section Divider
- Problem Page
- Diagnostic Page

- Matrix Page

- Comparison Page
- Process or Rollout Page
- Timeline Page
- Cards Page
- Chart Page
- Case Page
- Results and Next Steps
- Conclusion Page

## Selection Rules

- Use path pages when you need to show the overall implementation route.
- Use diagnostic pages when the page should explain where the real issue sits.
- Use decision pages when the user must understand why one choice comes first.
- Use matrix pages when value and feasibility must be balanced.
- Use cards pages for parallel points with equal weight.
- Use process or rollout pages when the order and handoff matter.
- Use chart pages only when the numeric view changes the decision.
- Use results and next-steps pages when the deck needs a management-style close.
